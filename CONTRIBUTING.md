# Contributing to NidhiSaarthi

Thank you for your interest in contributing to NidhiSaarthi! This document provides guidelines and instructions for contributing.

## Code of Conduct

Be respectful and inclusive. This project is dedicated to providing a welcoming environment for all contributors.

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create** a new branch for your feature: `git checkout -b feature/description`
4. **Make** your changes
5. **Test** your changes
6. **Commit** with clear messages: `git commit -m 'Add feature description'`
7. **Push** to your fork: `git push origin feature/description`
8. **Create** a Pull Request

## Development Setup

```bash
# Install dependencies
npm install

# Start dev server
npm run dev

# Run tests
npm run test

# Lint code
npm run lint
```

## Coding Standards

- **TypeScript**: Use strict mode, avoid `any` types
- **React**: Follow hooks patterns, functional components
- **Formatting**: Code should be automatically formatted by prettier
- **Testing**: Write tests for new features
- **Documentation**: Update docs for new/modified features

## Pull Request Process

1. Update README.md with details of changes if applicable
2. Ensure tests pass: `npm run test`
3. Ensure no linting errors: `npm run lint`
4. Include a clear PR description
5. Reference any related issues with `#issue-number`

## Reporting Bugs

Create an issue with:
- Clear, descriptive title
- Detailed description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Environment details (OS, Node version, etc.)

## Suggesting Enhancements

Submit an issue with:
- Clear description of the enhancement
- Use cases and benefits
- Possible implementation approach
